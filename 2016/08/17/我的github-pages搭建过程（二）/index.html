我的github-pages搭建过程（二）

  • github

切换hexo主题 and 绑定注册域名

  • 切换hexo主题

博客有了，也可以正常使用，但是默认的主题似乎太普通，想要一个自己喜欢的博客主题，可以去搜索并下载下来，我使用的是yilia主题。

1.首先在博客根目录运行git clone https://github.com/litten/hexo-theme-yilia themes/hexo-theme-yilia

或者直接将该主题下载下来，放到博客的themes目录下

2.然后修改_config.yml中的theme字段为hexo-theme-yilia，同样要注意冒号后面的空格符。

3.执行hexo g生成静态文件，可以在本地运行hexo s查看效果。执行hexo d将主题推送到github,等一分钟就可以看到新的主题了。

  • 绑定域名

每次访问博客都要填写https://xx.github.io是否觉得不够炫酷。
注册一个属于自己的域名，就可以通过注册的新域名访问自己的博客了。我是在万网上注册的，一般都不是很贵。

1.首先选一个没有被注册的域名，然后购买，购买之后要设置一下域名解析。（域名解析就是将IP地址与域名联系起来。我们访问百度的时候，都会输入www.baidu.com,而不会去访问14.215.177.37。我们在访问www.baidu.com的时候，网络会先去询问一个叫域名服务器的东西：“www.baidu.com的IP地址是多少呢？”这个时候，服务器告诉它：“14.215.177.37”，然后我们的电脑就去请求14.215.177.37的数据来达到访问百度的目的了。）如果不设置域名解析，那么访问新注册的域名就无法查看自己的博客了。

2.先查看自己博客的IP地址，window可以在cmd中运行ping xxx.github.io,图中的 151.101.16.133就是解析出来的IP地址

3.设置域名解析将IP地址与新域名联系起来，一般域名注册服务都支持设置域名解析，我是在万网上设置的，按照下面的方式配置。

4.等待3-5分钟后，再次在cmd中ping 新注册的的域名,IP地址和设置的一致，就说明成功了

5.在博客source/_posts/目录下添加一个叫CNAME不带后缀的文件,写上新注册的域名。运行hexo g -g推送到git,2分钟后，就可以使用新的域名访问自己的博客了。
