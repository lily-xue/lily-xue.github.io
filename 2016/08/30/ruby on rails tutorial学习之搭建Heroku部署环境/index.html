ruby on rails tutorial学习之搭建Heroku部署环境

Heroku是一个免费的rails应用程序托管平台，rails应用程序部署简单，下面来介绍托管方式。

本地部署环境

1.修改数据库

由于Heroku使用PostgreSQL,所以我们不能在生产环境使用sqlite了，只能在生产环境使用pg（用来与PostgreSQL通信），开发环境使用sqlite。

需要将Gemfile中的gem 'sqlite3'改为以下内容

然后运行bundle install --without production安装gem，这个命令不会在本地安装任何gem，但这步是必须的，因为要把pg的版本更新到Gemfile.lock文件中，这样Heroku才知道。

接下来提交上述改动到github，运行

git commit -a -m "Update Gemfile.lock for Heroku"

2.生成Heroku所需的资源文件,图片和CSS等

注册Heroku账号

1.在Heroku官网使用邮箱注册一个账号。

2.使用Heroku Toolbelt安装所需的软件。

3.在命令行下输入heroku登录命令：

$ heroku login

4.进入rails应用的根目录，使用heroku命令在heroku服务器上创建一个区域放置应用程序。heroku会自动创建一个子域名，可以立即生效。

5.通过git将应用程序推送到heroku中

$ git push heroku master

6.运行以下命令可以使用默认浏览器打开你的应用程序了

$ heroku open

7.以上为最基本的部署和使用方法，更多的命令可以参看heroku的官方文档。
