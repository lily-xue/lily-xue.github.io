我的github pages搭建过程（一）

  • github

从第一天开始接收到任务起，我就开始在搭建个人博客，在短短的半个月时间内，有至少6个晚上搭建博客到后半夜2点，心力交瘁。各种折腾各种自坑，在今天终于搭建好了，现在将相关教程和注意事项写下来，希望能够帮助到正在挣扎的你。

在搭建主页的过程中，我在网上搜索了很多教程基本上是小白式教程，没有写明每一步的原因，有时候漏掉一个步骤或者按照自己理解来做的时候———我管这个叫“自坑”，很可能会得到一个错误的结果。我反复看了教程好多遍才理解每一步的操作应该是怎么样的。自学者遇到的最大困难，就是明明看到胜利就在前方，缺少一个点破迷津的人，自己在不断挣扎，学习、吸取教训。

hexo安装与github安装 and 个人博客部署到github

我这个教程是基于window系统用户，Hexo3版本。
下面的这个教程涵盖了hexo安装和git注册建立仓库等操作，我就不一一描述。建议先按照这个教程安装一下，如果遇到问题再来看我这里是否有相关说明和解惑。
hexo干货系列：（一）hexo+gitHub搭建个人独立博客

1._config.yml中的aploy是将博客推送到github上的重要配置，如果这个字段错误，执行hexo d的时候会出错。

2._config.yml中的url字段配置为自己的github pages主页地址，比如我的是https://lily-xue.github.io

3.aploy配置的冒号后面有空格，type等字段前面也是有空格的，如果不正确会报错ERROR Deployer not found : github

4.在本地预览博客是否可以运行，然后使用hexo g生成静态文件，使用hexo d命令将博客推送到github。如果不生成静态文件，在git上就无法访问，出现404的报错了。

5.从始至终，是不需要使用git命令将这个hexo目录推送到github上的！

从始至终，是不需要使用git命令将这个hexo目录推送到github上的！

只需四步走：新建博客(hexo new “title”)，使用编辑source/_posts/下生成的新博客，生成静态文件(hexo g),推送到githug(hexo d)。

然后就可以看到自己新写的博客在github主页上显示了~
